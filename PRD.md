# Jag - Product Requirements Document

## Overview

Jag is a Chrome/Arc browser extension that intercepts distracting website visits with an AI-powered awareness moment. Instead of blocking sites entirely, it forces a brief, honest interaction: a personalized, context-aware message and a mandatory choice before proceeding. The extension talks to a local OpenClaw gateway via the OpenResponses API, giving it access to the user's schedule, values, patterns, and streak data, making every intervention unique and impossible to tune out.

## Problem Statement

Screen addiction follows the same neurological patterns as substance addiction: compulsive dopamine-seeking that fires automatically, bypasses intentions, and migrates across devices when blocked. Existing solutions fall into two categories that both fail for power users:

1. **Blockers** (Freedom, Screen Time): Work for fully restricted apps but fail for apps with legitimate uses (email, messaging). The "dumb phone problem": you can't block everything without becoming disconnected from the world.

2. **Friction tools** (one sec, long passwords, grayscale): Habituate within days. The brain learns the pattern and autopilots through it. one sec's research shows 57% reduction at population level, but high-craving users adapt faster.

**The gap:** No existing tool creates an awareness interrupt that is simultaneously (a) enforced and non-ignorable, (b) adaptive and anti-habituation, and (c) smart enough to distinguish legitimate from compulsive use.

## Design Philosophy

Grounded in James Clear's Atomic Habits inversion framework for breaking bad habits:

- **Law 1: Make it Invisible (Cue)** - Delete apps, use Freedom.to for hard blocks. Not part of the extension. Table stakes done separately.
- **Law 2: Make it Unattractive (Craving)** - The contextual mirror. AI-generated, personalized awareness that reframes the behavior by showing what you're trading off right now. Changes every time to prevent habituation.
- **Law 3: Make it Difficult (Response)** - Escalating timers that increase with frequency of access. Self-calibrating: legitimate use (open email once) gets a short pause; compulsive use (5th open in an hour) gets a long wait.
- **Law 4: Make it Unsatisfying (Reward)** - Streak tracking with visible loss. Breaking a multi-day streak creates immediate regret. Financial penalties deferred to v2.

**Key Design Principle:** Anything that relies on the user choosing to do the right thing in the moment of craving will fail. The system must enforce, not suggest.

## Target User

People who:
- Have tried multiple screen time solutions and habituated out of all of them
- Cannot fully block their devices due to legitimate work/life needs
- Recognize their behavior as addictive, not just a bad habit
- Have the self-awareness to set up constraints during rational moments

## Core User Flow

1. User navigates to a flagged site (e.g., reddit.com)
2. Full-page overlay appears immediately, blocking the site content
3. Overlay displays:
   - Contextual awareness line (AI-generated, unique each time)
   - Current streak status (e.g., "Day 7: your longest this month")
   - Session stats (e.g., "3rd time opening Reddit today")
   - Context-aware action buttons
4. User must select a button to proceed
5. Based on selection, an enforced timer counts down before the site loads
6. Site loads after timer completes (or user chose "Nevermind" and tab closes)

## Contextual Awareness Line

The single most important UI element. Generated by OpenClaw via the OpenResponses API. Examples:

- "You've been present for 4 hours straight, longest streak this week. Open Reddit?"
- "3rd time here in the last hour. You haven't gone for your run yet."
- "It's 9pm. Family time. This is the pattern you asked me to watch for."
- "Monday 10am. Pioneer Fund day. Inbox zero was your morning goal."

Key properties:
- Different every time (anti-habituation)
- Specific to the user's life, not generic
- States facts, doesn't lecture
- Includes temporal context (time of day, day of week, what should be happening)

## Context-Aware Buttons

Buttons adapt based on time of day, site category, and usage patterns.

**Reddit at 9pm Saturday:**
- "Just want to browse" (long timer)
- "Nevermind"
- *(No work option)*

**Email at 10am Monday:**
- "I need this for work" (short timer)
- "Just browsing" (longer timer)
- "Nevermind"

**Reddit at 2pm Wednesday, 4th time today:**
- "Just want to browse" (very long timer)
- "Nevermind"
- *(Work option removed for sites categorized as "never work")*

### Button Rules
- "Work" button only appears during configurable work hours
- "Work" button never appears for sites categorized as "never work" (Reddit, Instagram, etc.)
- Button selection is logged for weekly pattern review
- If user selects "Work" 5+ times/day for a non-work site, the weekly review surfaces this self-deception

## Escalating Timer System

Timers increase based on frequency within a rolling window.

**Base configuration:**
- 1st open in rolling window: 5 seconds
- 2nd open: 15 seconds
- 3rd open: 30 seconds
- 4th open: 60 seconds
- 5th+ open: 120 seconds

**Modifiers:**
- "Work" button selection: timer reduced by 50%
- "Just browsing" selection: timer at full value
- Outside work hours: base timers increased by 50%
- During configured "presence time" (family dinner, meditation): base timers doubled

Timer is enforced: no skip button, no closing the tab to bypass. Reopening the site triggers the same or higher tier.

Rolling window resets every 2 hours by default (configurable).

## Streak System

Daily streak: User sets a target (e.g., "under 30 min total on flagged sites per day"). Each day under target continues the streak.

**Display:**
- Current streak length prominently on every overlay
- Longest streak ever (creates aspiration)
- Streak resets to 0 on exceeding daily target

The streak serves double duty:
- **Law 2 (Unattractive):** Seeing "Day 14" makes opening Reddit feel like a threat
- **Law 4 (Unsatisfying):** Losing a streak creates immediate negative emotion

Insight from addiction research: Contingency management (escalating rewards for consecutive clean periods, with reset on relapse) is the single most evidence-backed treatment for stimulant addiction.

## Technical Architecture

### Extension Components
- **Content Script:** Detects navigation to flagged sites, injects full-page overlay
- **Background Service Worker:** Manages state (open counts, timers, streaks), communicates with OpenClaw
- **Overlay UI:** Renders the awareness screen with contextual line, stats, and buttons
- **Options Page:** Configure flagged sites, work hours, presence times, streak targets, API connection

### OpenClaw Integration
- **Endpoint:** `POST http://localhost:18789/v1/responses` (OpenResponses API)
- **Auth:** Bearer token (gateway shared secret)
- **Request:** Extension sends structured context (site, open count, time, streak, button history)
- **Response:** AI returns awareness text and button configuration as structured JSON
- **Fallback:** If gateway is unreachable, use locally cached context templates
- **Session:** Uses stable user ID for session continuity across calls

### Data Model (Chrome Local Storage)
- `sites`: { url_pattern, category, never_work, timer_multiplier }
- `sessions`: { site, timestamp, button_choice, timer_served }
- `streaks`: { current, longest, daily_minutes, target_minutes }
- `config`: { work_hours, presence_times, rolling_window_minutes }

### Privacy
- All data stored locally in Chrome storage
- No external servers (OpenClaw runs on localhost)
- No telemetry, no analytics
- Usage data never leaves the machine

## MVP Scope (Week 1)

### In scope
- Chrome extension with full-page overlay on flagged sites
- Configurable list of flagged sites (Reddit, Twitter/X, Instagram, YouTube, news sites)
- OpenClaw integration via OpenResponses API for contextual awareness line
- Context-aware button system (work/browse/nevermind) with adaptive display rules
- Escalating timer system based on open frequency
- Basic streak tracking (daily minutes on flagged sites)
- Simple options page for configuration

### Out of scope (future iterations)
- Phone app or cross-device sync
- Financial penalties (anti-charity donations)
- Live calendar integration (hardcode schedule initially)
- Conversation or chat interface with AI
- Weekly review or summary dashboard
- Social accountability features
- Reward system for streaks beyond the streak counter itself

## Success Metrics

**Week 1 (personal use):**
- Total daily minutes on flagged sites (target: 50% reduction from baseline)
- Ratio of "Nevermind" vs "Browse" selections (higher nevermind = working)
- Streak length (target: 5+ day streak within first 2 weeks)
- Subjective: Does it feel different from one sec? Does the contextual line actually register?

**Week 2+ (anti-habituation check):**
- Is the user still reading the awareness line, or glazing past it?
- Are timers still creating real friction, or just annoying pauses?
- Has the user attempted to disable or circumvent the extension?

## Open Questions

1. Should the overlay be bypassable with a keyboard shortcut for genuine emergencies?
2. What is the right rolling window for timer reset? (2 hours proposed)
3. Should "Nevermind" selections earn bonus streak points?
4. How to handle incognito/private browsing?
5. Should the extension track total time on flagged sites, or just intercept opens?
6. What is the right fallback when OpenClaw gateway is unreachable?
